从源代码编译安装——Makefile 1. 前言
  cmdtr提供了很多从源代码编译的方案，包括有Makefile、CMake以及安装脚本，用户可根据需要选择合适的方式编译源代码并安装。在开始之前，请确保你的操作系统中安装有gcc和CMake。在本文档提供的方法中，CMake为可选项。
  需要注意的是，在Windows下如果安装的是MinGW，则make命令需要改为mingw32-make。
  安装脚本可以一次性完成源代码的编译以及二进制文件的安装，成功安装的前提是该脚本是以root身份运行的。要想一次性完成编译和安装，只需要直接运行它。（Windows下无法使用此脚本）
2. 编译
  操作方法很简单，在Linux操作系统中只需要在源代码根目录下键入make命令即可，在Windows下需要指定特定平台的make文件，使用方法如下所示（如果无法使用make命令可以将其替换为mingw32-make或将其二进制文件的名称改为make）： 
make -f win32.make
  在编译完成后，源代码的根目录下会出现bin目录，如果没有出现编译、连接错误则会在该目录下生成libformula.so（.dll）和cmdtr（.exe）两个文件，这时就可以进行下一步的安装操作。
3. 安装
  安装过程与上一篇文档的方法差不多，请参照上一篇文档的第二部分。
